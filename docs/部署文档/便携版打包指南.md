## CPU 与 GPU 版本

小遥搜索提供两个版本的后端可执行文件：

| 特性 | CPU版本 | GPU版本 |
|------|---------|---------|
| **适用用户** | 大多数普通用户 | 有NVIDIA显卡的用户 |
| **PyTorch** | torch (CPU) | torch (CUDA 12.1) |
| **Faiss** | faiss-cpu | faiss-gpu |
| **打包大小** | ~150MB | ~200MB |
| **推理速度** | 基准 | 5-10倍加速 |
| **输出文件** | `xiaoyao-backend-cpu.exe` | `xiaoyao-backend-gpu.exe` |

## 快速开始

### CPU版本打包（推荐）

```bash
cd backend

# 确保安装了CPU版本的依赖
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cpu
pip install faiss-cpu

# 打包
pyinstaller xiaoyao_backend_cpu.spec
```

### GPU版本打包

```bash
cd backend

# 确保安装了GPU版本的依赖
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121
pip install faiss-gpu

# 打包
pyinstaller xiaoyao_backend_gpu.spec
```

## 打包流程

### 0. 快速测试（推荐！）

在完整打包前，建议先进行快速测试，以验证依赖配置是否正确。

**为什么需要快速测试？**
- 完整打包需要 30-60 分钟
- 每次修改配置后都要重新打包才能验证
- 快速测试只需 5-10 分钟即可验证基本配置

**快速测试流程：**

```bash
cd backend

# 方法1: 使用快速测试脚本（推荐）
quick_test.bat cpu    # 测试CPU版本
quick_test.bat gpu    # 测试GPU版本

# 方法2: 手动执行测试步骤
python check_missing.py      # 步骤1: 检查缺失依赖
pyinstaller --clean xiaoyao_backend_cpu.spec  # 步骤2: 打包
dist\xiaoyao-backend-cpu.exe # 步骤3: 测试运行
```

**测试工具说明：**

| 工具 | 用途 | 耗时 |
|------|------|------|
| `check_missing.py` | 检查常见缺失依赖（jaraco、pkg_resources等） | 5秒 |
| `analyze_deps.py` | 深度分析依赖，查看将被打包的内容 | 30秒 |
| `quick_test.bat` | 自动化快速测试流程 | 5-10分钟 |

**快速测试脚本功能：**
1. 检查常见缺失依赖
2. 清理旧的构建目录
3. 执行快速打包
4. 启动可执行文件验证

**判断测试是否成功：**
- ✅ 看到 `INFO: Uvicorn running on http://127.0.0.1:8000` → 配置正确
- ❌ 看到 `ModuleNotFoundError` → 需要在 spec 文件添加对应的 hiddenimport

### 1. 准备环境

```bash
# 安装 PyInstaller
pip install pyinstaller

# 安装项目依赖
cd backend
pip install -r requirements.txt
```

**CPU版本（大多数用户）：**

## 完整打包脚本

创建 `scripts/build-backend.bat`：

```batch
@echo off
echo ========================================
echo   小遥搜索后端打包脚本
echo ========================================

echo.
echo 请选择打包版本:
echo   1. CPU 版本（推荐大多数用户）
echo   2. GPU 版本（需要NVIDIA显卡）
echo.
set /p version="请输入选择 (1 或 2): "

cd backend

if "%version%"=="2" (
    echo.
    echo [*] 安装GPU版本依赖...
    pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121
    pip install faiss-gpu
    echo.
    echo [*] 打包GPU版本...
    pyinstaller xiaoyao_backend_gpu.spec
) else (
    echo.
    echo [*] 安装CPU版本依赖...
    pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cpu
    pip install faiss-cpu
    echo.
    echo [*] 打包CPU版本...
    pyinstaller xiaoyao_backend_cpu.spec
)

echo.
echo ========================================
echo   打包完成！
echo ========================================
echo.
echo 输出文件: backend\dist\
pause
```

## 常见问题

### Q0: 为什么需要快速测试？

**问题：** 每次修改配置都要等 30-60 分钟才能验证

**解决方案：** 使用快速测试工具
```bash
cd backend

# 运行快速测试（5-10分钟）
quick_test.bat cpu

# 或手动检查依赖
python check_missing.py
```

快速测试可以提前发现 90% 的依赖问题，避免浪费时间进行完整打包。

### Q1: 打包卡在 "Performing binary vs. data reclassification"

**解决方案：**
1. 已在 spec 文件中禁用 UPX 压缩 (`upx=False`)
2. 等待时间可能需要 10-30 分钟
3. 如果超过 30 分钟，按 Ctrl+C 中止并重试

### Q2: GPU版本打包后无法运行

**可能原因：**
- 目标机器没有 NVIDIA 显卡
- 没有安装 CUDA 驱动

**解决方案：**
- 让用户使用 CPU 版本
- 或安装 NVIDIA 驱动

### Q3: 打包文件体积过大

**当前大小：**
- CPU版本：~150MB
- GPU版本：~200MB

**如果需要减小：**
1. 排除更多不需要的模块（编辑 spec 文件的 `excludes`）
2. 使用 UPX 压缩（`upx=True`，但会增加打包时间）
