| **Ollama** | 最新版 | `runtime/ollama/` | 本地大语言模型（需安装） |

## 安装步骤

### 步骤1：解压整合包

```
1. 下载 XiaoyaoSearch-Windows-v1.0.0.zip
2. 解压到非系统盘目录（推荐）
   例如：D:\MyApps\XiaoyaoSearch-Windows-v1.0.0\
3. 确保目录路径不含中文或特殊字符
```

**解压后的目录结构：**

```
XiaoyaoSearch-Windows-v1.0.0/
├── runtime/                # 运行时环境（预置）
│   ├── python/
│   │   └── python-3.10.11-embed-amd64.zip  # Python嵌入式版本（解压即用）
│   ├── nodejs/            # Node.js运行时（便携版）
│   ├── ffmpeg/            # FFmpeg运行时（便携版）
│   └── ollama/            # Ollama安装包
├── backend/               # 后端源码
├── frontend/              # 前端源码
├── scripts/               # 自动化脚本
├── data/                  # 数据目录（首次运行创建）
└── README.txt             # 本说明文件
```

### 步骤2：运行环境准备脚本

```
1. 双击 setup.bat
2. 等待环境准备完成（首次约2-3分钟）
```

**环境准备脚本会执行以下操作：**

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 解压Python嵌入式版本 | 从zip解压到python-embed目录 |
| 2 | 检查运行时目录 | 验证runtime/目录完整性 |
| 3 | 配置环境变量 | 添加FFmpeg、Node.js、Python到PATH |
| 4 | 安装pip | 下载get-pip.py并安装到嵌入式版本 |
| 5 | 安装后端依赖 | 使用嵌入式Python安装requirements.txt |
| 6 | 安装前端依赖 | 使用预置Node.js运行时 |
| 7 | 生成配置文件 | 从.env.example生成.env |
| 8 | 创建数据目录 | 创建data/下必要目录 |
| 9 | 检查Ollama | 提示安装方式（本地/官网） |
| 10 | 检查AI模型 | 提供下载指引 |

**配置成功的标志：**

```
========================================
  环境准备完成！
========================================

✓ 运行时目录完整
✓ 环境变量已配置
✓ 后端依赖已安装
✓ 前端依赖已安装
✓ 配置文件已生成
✓ 数据目录已创建

下一步：双击 startup.bat 启动应用

提示：首次使用请先安装AI模型，详见模型下载章节
```

## 启动与停止

### 启动应用

**方式1：双击运行脚本（推荐）**

```
双击 startup.bat
```

运行脚本会自动：
1. 配置环境变量
2. 启动后端服务（使用预置Python虚拟环境）
3. 等待后端启动完成（自动检测，最多30秒）
4. 启动前端服务（使用预置Node.js运行时）
5. 自动打开浏览器访问应用

**启动方式说明：**
- 后端和前端会在独立的命令行窗口中运行
- 脚本会自动检测后端是否启动完成（通过HTTP请求或端口检查）
- 后端启动成功后才会启动前端

**方式2：手动启动**

```bash
# 启动后端（使用嵌入式Python）
cd runtime\python\python-embed
python.exe ..\..\..\backend\main.py

# 新开终端窗口，启动前端（使用预置Node.js）
cd runtime\nodejs
node.exe ..\..\frontend\node_modules\vite\bin\vite.js ..\..\frontend
```

### 访问应用

启动成功后，浏览器会自动打开：

| 服务 | 地址 | 说明 |
|------|------|------|
| **前端界面** | http://127.0.0.1:5173 | 主应用界面 |
| **后端API** | http://127.0.0.1:8000 | API根路径 |
| **API文档** | http://127.0.0.1:8000/docs | Swagger文档 |

### 停止应用

**停止方式：直接关闭窗口**

```bash
# 关闭"小遥搜索-后端"和"小遥搜索-前端"两个命令行窗口
# 或者在窗口中按 Ctrl+C
```

**说明：**
- 后端和前端在独立窗口中运行，直接关闭窗口即可停止
- 无需额外的停止脚本，操作简单直接

## 常见问题

### Q1: 环境准备失败

**问题：** setup.bat 执行失败

**解决方案：**

```bash
# 1. 检查运行时目录
确保 runtime/ 目录存在且包含：
- python/python-3.10.11-embed-amd64.zip
- nodejs/
- ffmpeg/
- ollama/

# 2. 检查网络连接
确保可以访问 pypi.tuna.tsinghua.edu.cn 和 bootstrap.pypa.io

# 3. 手动安装依赖
如果自动安装失败，可手动安装：

# 安装后端依赖
cd runtime\python\python-embed
python.exe -m pip install -r ..\..\..\backend\requirements.txt

# 安装前端依赖
cd runtime\nodejs
node.exe ..\..\frontend\node_modules\npm\bin\npm-cli.js install ..\..\frontend
```

### Q2: 后端启动失败

**问题：** startup.bat 启动后端失败

**解决方案：**

```bash
# 1. 检查嵌入式Python
确保 runtime/python/python-embed/ 目录存在

# 2. 手动启动后端查看错误
cd runtime\python\python-embed
python.exe ..\..\..\backend\main.py

# 3. 检查端口占用
netstat -ano | findstr :8000
如果有占用，关闭占用进程或修改 backend/.env 中的端口
```

### Q3: 前端启动失败

**问题：** startup.bat 启动前端失败

**解决方案：**

```bash
# 1. 检查 Node 依赖
确保 frontend/node_modules/ 目录存在

# 2. 重新安装依赖
cd runtime\nodejs
node.exe ..\..\frontend\node_modules\npm\bin\npm-cli.js install ..\..\frontend

# 3. 手动启动前端查看错误
cd runtime\nodejs
node.exe ..\..\frontend\node_modules\vite\bin\vite.js ..\..\frontend
```

### Q4: 模型加载失败

**问题：** 启动后提示模型加载失败

**解决方案：**

```bash
# 1. 检查模型路径
确保模型文件在正确的目录：

# 嵌入模型
data/models/embedding/BAAI/bge-m3/

# 语音识别模型
data/models/faster-whisper/Systran/faster-whisper-base/

# 视觉模型
data/models/cn-clip/OFA-Sys/chinese-clip-vit-base-patch16/

# 2. 检查配置文件
访问「系统设置」→「模型配置」：
- 确认模型路径正确
- 点击「测试连接」验证模型
- 切换到已下载的模型版本

# 3. 查看后端日志
检查 data/logs/backend.log 获取详细错误信息
tail -n 50 data/logs/backend.log

# 4. 常见错误处理

错误：「找不到模型文件」
解决：检查模型路径是否正确，确保模型已下载

错误：「模型加载超时」
解决：切换到更小的模型版本，或增加系统内存

错误：「Ollama连接失败」
解决：确保Ollama服务正在运行（ollama serve）
```

### Q5: FFmpeg 不可用

**问题：** 提示找不到 FFmpeg

**解决方案：**

```bash
# 1. 检查 FFmpeg 运行时
确保 runtime/ffmpeg/ 目录存在且包含 ffmpeg.exe

# 2. 手动配置环境变量
set PATH=%PATH%;%CD%\runtime\ffmpeg

# 3. 验证 FFmpeg
runtime\ffmpeg\ffmpeg.exe -version
```

### Q6: Node.js 不可用

**问题：** 提示找不到 Node.js

**解决方案：**

```bash
# 1. 检查 Node.js 运行时
确保 runtime/nodejs/ 目录存在且包含 node.exe

# 2. 手动配置环境变量
set PATH=%PATH%;%CD%\runtime\nodejs

# 3. 验证 Node.js
runtime\nodejs\node.exe --version
```

### Q7: 端口被占用

**问题：** 8000 或 5173 端口被占用

**解决方案：**

```bash
# 1. 查找占用进程
netstat -ano | findstr :8000

# 2. 结束占用进程
taskkill /PID <进程ID> /F

# 3. 或修改配置
编辑 backend/.env，修改后端端口：
backend:
  port: 8001  # 改为其他端口
```

### Q8: Ollama 安装问题

**问题：** Ollama 安装失败或无法启动

**解决方案：**

```bash
# 1. 手动安装 Ollama
双击 runtime/ollama/ 目录下的安装包

# 2. 验证安装
ollama --version

# 3. 启动 Ollama 服务
ollama serve

# 4. 下载模型
ollama pull qwen2.5:7b
```

### Q9: 杀毒软件误报

**问题：** 杀毒软件阻止运行

**解决方案：**

```bash
# 1. 添加白名单
将小遥搜索目录添加到杀毒软件白名单

# 2. 临时关闭杀毒软件
在安装配置期间临时关闭

# 3. 使用 Windows Defender
推荐使用 Windows Defender，误报率低
```

### 脚本说明

**setup.bat** - 环境准备脚本

```
功能：
- 解压Python嵌入式版本（python-3.10.11-embed-amd64.zip）
- 配置python310._pth以支持pip
- 下载并安装get-pip.py
- 检查运行时目录完整性
- 配置环境变量（FFmpeg、Node.js、Python）
- 安装后端Python依赖（使用requirements.txt）
- 安装前端Node依赖（使用package.json）
- 生成配置文件backend/.env（从.env.example复制）
- 创建数据目录结构
- 检查Ollama安装状态（提示本地安装包+官网）
- 检查AI模型下载状态（提供下载指引）

执行时机：首次运行或运行时更新后
```

**startup.bat** - 启动脚本

```
功能：
- 配置环境变量
- 启动后端服务（使用runtime/python/python-embed/python.exe）
- 等待后端启动完成（循环检测HTTP或端口，最多30秒）
- 启动前端服务（使用runtime/nodejs/node.exe）
- 自动打开浏览器访问应用

执行时机：每次启动应用

停止方式：直接关闭"小遥搜索-后端"和"小遥搜索-前端"窗口
```

### 配置文件说明

**backend/.env** - 后端环境变量（从.env.example生成）

```bash
# 数据目录
DATA_DIR=./data

# 运行时配置
PYTHON_PATH=../runtime/python/venv/Scripts/python.exe
FFMPEG_PATH=../runtime/ffmpeg/bin/ffmpeg.exe

# 索引配置
FAISS_INDEX_PATH=../data/indexes/faiss
WHOOSH_INDEX_PATH=../data/indexes/whoosh
DATABASE_PATH=../data/database/xiaoyao_search.db

# API配置
API_HOST=127.0.0.1
API_PORT=8000
API_RELOAD=true

# 日志配置
LOG_LEVEL=info
LOG_FILE=../data/logs/app.log

# 模型配置
# 嵌入模型路径
EMBEDDING_MODEL_PATH=../data/models/embedding
EMBEDDING_MODEL_NAME=BAAI/bge-m3

# 语音识别模型路径
WHISPER_MODEL_PATH=../data/models/faster-whisper
WHISPER_MODEL_NAME=Systran/faster-whisper-base

# 视觉模型路径
CLIP_MODEL_PATH=../data/models/cn-clip
CLIP_MODEL_NAME=OFA-Sys/chinese-clip-vit-base-patch16

# Ollama配置
OLLAMA_HOST=http://127.0.0.1:11434
OLLAMA_MODEL=qwen2.5:1.5b
```

**模型配置说明**：

在「系统设置」→「模型配置」界面可以调整以下设置：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| **嵌入模型** | bge-m3 | 影响搜索准确度 |
| **语音识别** | faster-whisper-base | 影响转录速度和准确度 |
| **视觉模型** | chinese-clip-vit-base | 影响图像理解能力 |
| **大语言模型** | qwen2.5:1.5b | 影响文本生成质量 |

**性能调优建议**：

- **追求速度**：使用 small/base 版本模型
- **追求准确度**：使用 medium/large 版本模型
- **资源受限**：降低模型大小，关闭不需要的功能
- **专业场景**：使用 large 版本模型 + GPU 加速
