# 架构决策 - 插件化架构与语雀数据源

**决策日期**：2026年2月22日

## 背景

小遥搜索MVP开发已完成，当前仅支持本地文件系统作为数据源。根据路线图规划，需要扩展多数据源支持（语雀、飞书、钉钉、Notion等），以提升产品竞争力和用户体验。

**当前痛点**：
- 数据源耦合：各数据源直接集成在核心代码中，扩展新数据源需要修改核心代码
- 开发效率低：每次添加新数据源都需要重复实现相似的扫描、解析、索引逻辑
- 社区贡献困难：第三方开发者无法独立开发和贡献数据源插件

**目标**：
- 建立插件化架构，实现数据源的热插拔和独立开发
- 优先实现语雀数据源作为第一个插件案例
- 为后续飞书、钉钉、Notion等数据源提供标准化的开发范式

## 解决方案

### 架构选择：插件化架构（方案二）

采用**插件化架构**而非微服务化架构，基于以下考虑：

| 对比项 | 插件化架构 | 微服务化架构 |
|--------|-----------|-------------|
| 实现周期 | 4-6周 | 8-10周 |
| 复杂度 | 中等 | 高 |
| 运维成本 | 低 | 高 |
| 适合团队规模 | 1-10人 | 10+人 |
| 符合当前需求 | ✅ | ❌ |

### 核心设计

#### 1. 插件接口定义

```
backend/app/plugins/interface.py
├── PluginMetadata          # 插件元数据（id、name、version等）
├── DataSourcePlugin        # 数据源插件接口
│   ├── get_metadata()      # 获取插件元数据
│   ├── get_config_schema() # 获取配置schema
│   ├── initialize()        # 初始化插件
│   ├── scan()              # 扫描数据项
│   ├── get_content()       # 获取内容
│   └── 生命周期方法         # install/uninstall/enable/disable
```

#### 2. 插件管理器

```
backend/app/plugins/manager.py
├── discover_plugins()      # 扫描插件目录
├── load_plugin()           # 动态加载插件
├── enable_plugin()         # 启用插件（热插拔）
├── disable_plugin()        # 禁用插件
└── get_plugin()            # 获取插件实例
```

#### 3. 插件目录结构

```
data/plugins/
├── yuque/                  # 语雀插件
│   ├── plugin.py           # 插件入口
│   ├── config.yaml         # 配置schema
│   ├── requirements.txt    # 依赖
│   └── README.md           # 说明文档
└── plugin_states.json      # 插件启用状态
```

### 语雀数据源插件设计

#### API对接

- **认证方式**: API Token（个人访问令牌）
- **API端点**: https://www.yuque.com/api/v2
- **支持资源**: 文档、知识库、用户信息

#### 数据流向

```
语雀知识库 → API调用 → 文档列表/内容
                            ↓
                    DataSourceItem标准化
                            ↓
                    统一索引服务
                            ↓
                    Faiss + Whoosh
```

#### 配置参数

```yaml
api_token: str              # API Token（必填）
repo_slug: str             # 知识库标识（必填，格式：用户名/知识库名）
base_url: str              # API端点（可选，默认官方）
sync_interval: int         # 同步间隔（可选，默认60分钟）
```

### API接口设计

```
POST /api/plugins/upload              # 上传插件包
GET  /api/plugins/list                # 列出所有插件
POST /api/plugins/{id}/enable         # 启用插件
POST /api/plugins/{id}/disable        # 禁用插件
GET  /api/plugins/{id}/config/schema  # 获取配置schema
POST /api/datasources/scan            # 扫描数据源
GET  /api/datasources/status          # 查询同步状态
```

> **说明**：本特性专注于后端插件架构，配置和管理通过API接口完成，暂不包含前端管理界面。

### 依赖关系

```
插件框架 → 语雀插件实现 → 索引集成 → API接口 → API文档
  (1周)      (1-2周)       (3-5天)     (3-5天)    (同步)
```

### 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 动态加载稳定性 | 中 | 完善插件验证机制、错误隔离 |

| 插件安全管理 | 中 | 代码沙箱、权限控制 |

第三方开发者可参考语雀插件实现，独立开发和贡献新的数据源插件。
