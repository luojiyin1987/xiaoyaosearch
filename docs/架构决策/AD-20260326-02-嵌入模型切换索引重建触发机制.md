# 架构决策 - 嵌入模型切换索引重建触发机制

**决策日期**：2026年3月26日
**相关决策**：AD-20260326-01（云端嵌入模型与索引重建方案）

## 背景

用户在设置页面切换嵌入模型后，系统需要智能检测并提示用户是否需要重建索引。根据不同场景，重建的紧迫性不同：

- **维度不匹配**（1024维 → 1536维）：强制重建，语义搜索完全失效
- **维度相同但模型不同**（BGE-M3 → text-embedding-v3）：建议重建，搜索质量下降
- **相同提供商换模型**：建议重建，向量空间可能不同

## 核心决策

### 决策一：自动检测 + 智能提示

**决策**：在用户保存嵌入模型配置时，后端自动检测模型变化，返回不同的重建建议级别。

**重建级别定义**：

| 级别 | 条件 | 行为 | 用户体验 |
|------|------|------|----------|
| `mandatory` | 向量维度不同 | 强制重建 | 显示模态框，阻止操作，必须重建 |
| `recommended` | 提供商或模型名不同 | 建议重建 | 显示通知，提供重建按钮，可延后 |
| `none` | 配置未变化 | 无需重建 | 正常保存 |

**后端实现**：

```python
# backend/app/api/ai_model_config.py

async def update_embedding_model_config(request: AIModelConfigRequest):
    """更新嵌入模型配置，自动检测是否需要重建索引"""

    # 1. 获取当前激活的嵌入模型
    old_model = await get_active_embedding_model()
    old_provider = old_model.provider
    old_model_name = old_model.model_name
    old_dimension = await detect_model_dimension(old_model)

    # 2. 保存新配置
    new_model = await save_model_config(request)
    new_provider = request.provider
    new_model_name = request.model_name

    # 3. 检测新模型的向量维度
    new_dimension = await detect_model_dimension(new_model)

    # 4. 判断场景并返回重建建议
    if old_dimension != new_dimension:
        # 场景A：维度不匹配，强制重建
        return {
            "success": True,
            "message": "嵌入模型配置已更新",
            "data": {
                "requires_rebuild": True,
                "rebuild_level": "mandatory",
                "rebuild_reason": f"向量维度不匹配：{old_dimension}维 → {new_dimension}维",
                "rebuild_impact": "语义搜索将完全失效，必须重建索引",
                "estimated_time": estimate_rebuild_time()
            }
        }

    elif old_provider != new_provider or old_model_name != new_model_name:
        # 场景B：维度相同但模型不同，建议重建
        return {
            "success": True,
            "message": "嵌入模型配置已更新",
            "data": {
                "requires_rebuild": True,
                "rebuild_level": "recommended",
                "rebuild_reason": "嵌入模型已更改，建议重建索引以获得最佳搜索质量",
                "rebuild_impact": "当前索引使用旧模型的向量空间，搜索结果可能不准确",
                "estimated_time": estimate_rebuild_time()
            }
        }

    else:
        # 场景C：配置未实际变化
        return {
            "success": True,
            "message": "嵌入模型配置已更新",
            "data": {
                "requires_rebuild": False
            }
        }
```

**前端响应式处理**：

```typescript
// frontend/src/renderer/src/views/Settings.vue

async function handleSaveEmbeddingConfig() {
  loading.value = true

  try {
    const response = await updateAIModelConfig(embeddingConfig)

    if (response.data.requires_rebuild) {
      const { rebuild_level, rebuild_reason, estimated_time } = response.data

      if (rebuild_level === 'mandatory') {
        // 强制重建：显示模态框
        Modal.confirm({
          title: '需要重建索引',
          content: `${rebuild_reason}\n\n${response.data.rebuild_impact}\n\n预计耗时：${estimated_time}`,
          okText: '立即重建',
          cancelText: '取消',
          onOk: () => startRebuildIndex(),
          onCancel: () => revertModelConfig()
        })
      } else if (rebuild_level === 'recommended') {
        // 建议重建：显示通知（带操作按钮）
        notification.warning({
          message: '嵌入模型已更改',
          description: `${rebuild_reason}\n${response.data.rebuild_impact}`,
          duration: 0, // 不自动关闭
          btn: (h) => h(Button, {
            type: 'primary',
            size: 'small',
            onClick: () => startRebuildIndex()
          }, '重建索引')
        })
      }
    } else {
      message.success('嵌入模型配置已更新')
    }
  } catch (error) {
    message.error('配置更新失败')
  } finally {
    loading.value = false
  }
}
```

### 决策三：搜索时检测（兜底机制）

**决策**：在用户执行搜索时，如果检测到索引和模型不匹配，显示警告横幅。

**实现位置**：搜索结果页面顶部

## 用户体验流程

### 流程 A：维度不匹配（强制重建）

```
1. 用户切换嵌入模型（BGE-M3 → text-embedding-3-small）
   ↓
2. 点击"保存配置"
   ↓
3. 后端检测到维度变化（1024 → 1536）
   ↓
4. 前端显示强制重建模态框：
   "向量维度不匹配，语义搜索将完全失效。
    必须重建索引才能继续使用。"
   ↓
5. 用户选择：
   - "立即重建" → 开始重建流程
   - "取消" → 回滚到原模型配置
```

### 流程 B：维度相同但模型不同（建议重建）

```
1. 用户切换嵌入模型（BGE-M3 → text-embedding-v3）
   ↓
2. 点击"保存配置"
   ↓
3. 后端检测到模型变化，但维度相同（1024 → 1024）
   ↓
4. 前端显示建议重建通知：
   "嵌入模型已更改，建议重建索引以获得最佳搜索质量。
    [重建索引]"  （可关闭）
   ↓
5. 用户选择：
   - 点击"重建索引"按钮 → 开始重建
   - 关闭通知 → 继续使用（搜索质量可能下降）
   ↓
6. 如果用户选择不重建，后续搜索时显示警告横幅：
   "⚠️ 当前索引使用旧模型创建，搜索结果可能不准确。[重建索引]"
```

### 流程 C：手动触发重建

```
1. 用户在设置页面点击"重建索引"按钮
   ↓
2. 显示重建确认模态框：
   "重建索引将：
    - 备份当前索引
    - 使用当前嵌入模型重建
    - 预计耗时 X 分钟

    是否继续？"
   ↓
3. 用户确认 → 开始重建
   ↓
4. 显示重建进度模态框（实时更新）
   ↓
5. 重建完成 → 关闭模态框
```

## 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 用户忽略建议重建 | 中 | 搜索时显示警告横幅，持续提示 |
| 重建时间估算不准 | 低 | 显示实时进度，支持取消 |
| 重建期间用户搜索 | 低 | 重建不影响旧索引，可正常搜索 |
| 网络中断导致重建失败 | 中 | 自动回滚，保留原索引 |
