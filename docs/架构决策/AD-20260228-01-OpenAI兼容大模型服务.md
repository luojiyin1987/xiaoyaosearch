# 架构决策 - OpenAI兼容大模型服务

**决策日期**：2026年2月28日

## 背景

小遥搜索当前支持 Ollama 本地大语言模型，用于查询增强等功能。随着 AI 市场发展，越来越多的云服务提供商提供了兼容 OpenAI API 标准的大语言模型服务（如阿里云通义千问、DeepSeek、Moonshot 等）。

**当前痛点**：
- 仅支持本地 Ollama，对硬件有要求，用户体验受限
- 云端备选方案已预留配置但缺少实现（`ollama_service.py:366-380`）
- 无法利用云端大模型的更强能力和更低成本

**目标**：
- 支持主流云服务提供商的大语言模型
- 提供统一的配置界面，简化用户配置流程
- 保持本地 Ollama 支持，用户可自由选择

## 解决方案

### 核心设计：类型互斥的模型配置

采用**类型互斥**的设计，用户在前端配置时只能选择一种类型：Ollama（本地）或 OpenAI 兼容（云端）。

```
大语言模型配置
├── 类型选择
│   ├── Ollama（本地）
│   │   ├── 模型名称：qwen2.5:1.5b
│   │   └── 服务地址：http://localhost:11434
│   └── OpenAI兼容（云端）
│       ├── API密钥：sk-xxxxx
│       ├── 端点地址：https://dashscope.aliyuncs.com/compatible-mode/v1
│       └── 模型名称：qwen-turbo
```

### 架构设计

#### 1. 服务类设计

```
backend/app/services/
├── ai_model_base.py              # 基类（已存在）
├── ollama_service.py             # Ollama 本地服务（精简）
└── openai_llm_service.py         # OpenAI 兼容服务（新建）
```

#### 2. 统一的接口规范

所有 LLM 服务继承自 `BaseAIModel`，实现统一接口：

```python
class BaseAIModel(ABC):
    @abstractmethod
    async def load_model() -> bool
    @abstractmethod
    async def predict(messages, **kwargs) -> Dict[str, Any]
    @abstractmethod
    def get_model_info() -> Dict[str, Any]
```

#### 3. 配置结构

**Ollama 本地配置**：
```json
{
  "model_type": "llm",
  "provider": "local",
  "model_name": "qwen2.5:1.5b",
  "config": {
    "base_url": "http://localhost:11434",
    "timeout": 30,
    "temperature": 0.7
  }
}
```

**OpenAI 兼容云端配置**：
```json
{
  "model_type": "llm",
  "provider": "cloud",
  "model_name": "qwen-turbo",
  "config": {
    "api_key": "sk-xxxxx",
    "endpoint": "https://dashscope.aliyuncs.com/compatible-mode/v1",
    "timeout": 60,
    "temperature": 0.7,
    "max_tokens": 2048
  }
}
```

### 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| API 密钥安全 | 高 | 敏感信息加密存储、不上传云端 |
| 网络依赖 | 中 | 本地 Ollama 作为备选、离线模式支持 |
| 成本控制 | 低 | 添加使用量统计、警告提示 |
