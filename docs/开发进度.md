## 🎬 视频画面搜索（已暂停 ⏸️）

> **暂停说明**: 本功能已完成需求分析和技术方案设计，因项目资源调整暂停开发，待后续时机恢复。

### 需求状态 ✅
- **产品定位**: 为小遥搜索添加视频画面搜索能力，用户可通过图片检索视频内容
- **目标用户**: 内容创作者（视频博主、UP主）、知识工作者（教师、学生、研究人员）
- **核心价值**:
  - 扩展搜索能力覆盖视频内容，提升产品竞争力
  - 弥补现有视频搜索仅基于音频转录的不足
  - 为内容创作者和知识工作者提供更高效的检索工具

### 技术方案 ✅
- **关键技术栈**:
  - FFmpeg: 视频关键帧提取（2倍于OpenCV性能）
  - CN-CLIP: 图像特征编码（复用现有图搜图能力）
  - Faiss: 向量相似度搜索（与图搜图共享索引）
  - SQLite: video_frames表存储帧元数据
- **功能开关设计**: 默认禁用，通过后端.env配置启用
- **配置参数**:
  - VIDEO_FRAME_SEARCH_ENABLED: 是否启用
  - VIDEO_FRAME_INTERVAL: 关键帧提取间隔（默认10秒）
  - VIDEO_FRAME_MAX_DURATION: 最大处理时长（默认600秒）
  - VIDEO_FRAME_RESOLUTION: 帧分辨率（默认224像素）

## 🔌 插件化架构与语雀数据源（规划中 🚧）

> **规划说明**: 本功能已完成需求分析和技术方案设计，进入待开发阶段，预计4-6周开发周期。

### 需求状态 ✅
- **产品定位**: 建立插件化架构，实现数据源的热插拔和独立开发，优先支持语雀知识库数据源
- **目标用户**: 知识工作者（产品经理、开发者、运维人员）、第三方开发者
- **核心价值**:
  - 扩展产品边界，从本地搜索升级为全场景知识搜索
  - 提升用户粘性，云端数据源绑定增强平台依赖
  - 建立插件生态，降低后续数据源接入成本

### 技术方案 ✅
- **关键技术栈**:
  - Python ABC: 插件接口定义（标准库，类型安全）
  - importlib: 插件动态加载（Python标准库，无额外依赖）
  - Pydantic: 配置验证（已有依赖，schema验证）
  - httpx: 语雀API调用（异步HTTP客户端，已集成）
  - PyYAML: 插件配置（人类可读，易于编辑）
- **架构设计**: 四层架构（API层 → 服务层 → 插件层 → 数据层）
- **核心接口**: `DataSourcePlugin` 抽象基类定义插件契约
- **数据源支持**: 本地文件系统（已有）、语雀知识库（P0）、飞书/钉钉（P1）
