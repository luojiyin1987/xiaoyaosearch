## 设计原则：零配置 + JSON元数据解析 + 复用现有架构

本特性基于**插件化架构**（由 plugins+yuque 特性建立），钉钉数据源采用**零配置元数据文件解析**方案：
- **复用现有插件基础设施**：BasePlugin、DataSourcePlugin、PluginLoader
- **无需配置文件**：自动识别 .xyddjson 元数据文件
- **无需外部同步**：下载工具自动生成元数据文件，无 API 调用
- **性能优化**：独立元数据文件，解析速度快于内容解析方案
- **完全兼容**：前端、数据库、索引服务均已支持

**核心逻辑**：
```python
def get_file_source_info(self, file_path: str, content: str = None) -> Dict[str, Any]:
    """
    获取文件的数据源信息

    1. 检查是否存在同名 .xyddjson 文件
    2. 读取并解析 JSON 元数据
    3. 验证 source 字段是否为 'dingtalk-docs'
    4. 提取 source_type、source_url、source_id
    5. 返回结果或 None（非钉钉文档）
    """
    # 读取元数据文件
    metadata = self._read_metadata_file(file_path)

    if not metadata:
        return {"source_type": None, "source_url": None, "source_id": None}

    # 提取关键字段
    file_info = metadata.get("file", {})

    return {
        "source_type": "dingding",
        "source_url": file_info.get("url"),
        "source_id": file_info.get("dentryUuid"),
        "export_time": metadata.get("exportTime"),
        "original_name": file_info.get("originalName")
    }

def _read_metadata_file(self, file_path: str) -> Optional[Dict[str, Any]]:
    """
    读取 .xyddjson 元数据文件

    1. 构造元数据文件路径：file_path + '.xyddjson'
    2. 检查文件是否存在
    3. 读取并解析 JSON
    4. 验证 source 字段
    5. 返回元数据字典或 None
    """
    metadata_path = Path(str(file_path) + '.xyddjson')

    if not metadata_path.exists():
        return None

    try:
        with open(metadata_path, 'r', encoding='utf-8') as f:
            metadata = json.load(f)

        if metadata.get('source') != 'dingtalk-docs':
            return None

        return metadata
    except Exception as e:
        logger.warning(f"读取 .xyddjson 失败: {e}")
        return None
```

# 钉钉文档插件

识别钉钉导出工具生成的 .xyddjson 元数据文件并提取文档信息。

## 使用方法

1. 安装钉钉导出浏览器插件

3. 将导出的文件放置到扫描目录

**Q: 元数据解析失败怎么办？**
A: 解析失败不影响文件索引，只是不会显示"钉钉"标识。
