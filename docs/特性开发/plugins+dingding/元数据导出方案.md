# 元数据导出方案

## 目的

导出钉钉文档时，为每个文件生成对应的元数据文件，方便小遥搜索识别文件来源和建立与云端文档的映射关系。

## 数据结构

### 单文件元数据格式

**文件名：** `项目报告.docx.xyddjson`

```json
{
  "version": "1.0.0",
  "source": "dingtalk-docs",
  "exportTool": "xiaoyaosearch-dingding-export",
  "exportToolVersion": "1.0.0",
  "exportTime": "2026-04-08T15:30:00.000Z",
  "file": {
    "fileName": "项目报告.docx",
    "originalName": "项目报告",
    "dentryUuid": "A1B2C3D4E5F6G7H8",
    "dentryKey": "key_abc123",
    "docKey": "doc_xyz789",
    "url": "https://alidocs.dingtalk.com/i/nodes/A1B2C3D4E5F6G7H8",
    "fileSize": 24576,
    "extension": "adoc",
    "exportFormat": ".docx",
    "contentType": "alidoc"
  }
}
```

### 字段说明

| 字段 | 说明 | 示例 |
|------|------|------|
| `version` | 元数据格式版本 | `"1.0.0"` |
| `source` | 数据来源 | `"dingtalk-docs"` |
| `exportTool` | 导出工具名称 | `"xiaoyaosearch-dingding-export"` |
| `exportToolVersion` | 导出工具版本 | `"1.0.0"` |
| `exportTime` | 导出时间 (ISO 8601) | `"2026-04-08T15:30:00.000Z"` |
| `file.fileName` | 导出后的文件名 | `"项目报告.docx"` |
| `file.originalName` | 钉钉文档原始名称 | `"项目报告"` |
| `file.dentryUuid` | 钉钉文档唯一ID | `"A1B2C3D4E5F6G7H8"` |
| `file.dentryKey` | 钉钉文档密钥 | `"key_abc123"` |
| `file.docKey` | 文档密钥 | `"doc_xyz789"` |
| `file.url` | 钉钉文档访问地址 | `"https://alidocs.dingtalk.com/i/nodes/xxx"` |
| `file.fileSize` | 文件大小（字节） | `24576` |
| `file.extension` | 原始扩展名 | `"adoc"` |
| `file.exportFormat` | 导出格式 | `".docx"` |
| `file.contentType` | 内容类型 | `"alidoc"` |

## 实现方案

### 设计选择：同名附属文件

**选择理由：**
1. ✅ 元数据和文件紧密关联，一目了然
2. ✅ 移动/重命名文件时，用户知道要同时操作 `.xyddjson`
3. ✅ 小遥搜索导入时，直接读取同目录的 `.xyddjson`
4. ✅ 简单直观，易于理解和维护

---

// ============================================
// 写入单文件元数据 - 用于小遥搜索识别
// ============================================
async writeMetadataFile(dirHandler, fileName, exportTime) {
    const metadata = {
        version: "1.0.0",
        source: "dingtalk-docs",
        exportTool: "xiaoyaosearch-dingding-export",
        exportToolVersion: version,
        exportTime: exportTime,
        file: {
            fileName: fileName,
            originalName: this.dentryInfo.name,
            dentryUuid: this.dentryInfo.dentryUuid,
            dentryKey: this.dentryInfo.dentryKey,
            docKey: this.dentryInfo.docKey,
            url: `https://alidocs.dingtalk.com/i/nodes/${this.dentryInfo.dentryUuid}`,
            fileSize: this.dentryInfo.fileSize,
            extension: this.dentryInfo.extension,
            exportFormat: newext,
            contentType: this.dentryInfo.contentType
        }
    };

    const metadataFileName = `${fileName}.xyddjson`;
    const metadataHandle = await dirHandler.getFileHandle(metadataFileName, {create: true});

    const writable = await metadataHandle.createWritable();
    await writable.write(JSON.stringify(metadata, null, 2));
    await writable.close();
}

## 小遥搜索使用方式

### 1. 读取文件元数据

```javascript
import fs from 'fs';

// 读取文件时查找同名元数据
function readMetadata(filePath) {
    const metadataPath = filePath + '.xyddjson';

    if (fs.existsSync(metadataPath)) {
        return JSON.parse(fs.readFileSync(metadataPath, 'utf-8'));
    }

    return null;
}

// 使用示例
const filePath = 'D:\\文档\\项目报告.docx';
const metadata = readMetadata(filePath);

if (metadata) {
    console.log('原始文档:', metadata.file.originalName);
    console.log('钉钉链接:', metadata.file.url);
    console.log('导出时间:', metadata.exportTime);
}
```

### 2. 建立索引

```javascript
// 扫描目录建立索引
function buildIndex(dirPath) {
    const fileMap = new Map();

    function scanDir(path) {
        const files = fs.readdirSync(path);

        files.forEach(file => {
            const fullPath = `${path}/${file}`;

            // 跳过 .xyddjson 文件
            if (file.endsWith('.xyddjson')) {
                return;
            }

            const stat = fs.statSync(fullPath);

            if (stat.isDirectory()) {
                scanDir(fullPath);
            } else {
                // 读取元数据
                const metadata = readMetadata(fullPath);

                if (metadata) {
                    fileMap.set(metadata.file.dentryUuid, {
                        path: fullPath,
                        metadata: metadata
                    });
                }
            }
        });
    }

    scanDir(dirPath);
    return fileMap;
}
```

### 3. 匹配云端文档

```javascript
// 用户从钉钉复制文档链接
const dingtalkUrl = "https://alidocs.dingtalk.com/i/nodes/xxx";
const dentryUuid = dingtalkUrl.split('/').pop();

// 查找本地文件
const index = buildIndex('D:\\我的文档\\钉钉导出');
const localFile = index.get(dentryUuid);

if (localFile) {
    console.log(`找到本地文件: ${localFile.path}`);
    console.log(`导出格式: ${localFile.metadata.file.exportFormat}`);
    console.log(`原始名称: ${localFile.metadata.file.originalName}`);
}
```

## 扩展名说明

### .xyddjson 格式

- **含义**: XiaoYao JSON = 小遥搜索专属 JSON 格式
- **目的**: 标识这是小遥搜索的元数据文件
- **优势**:
  - 避免被小遥搜索当作普通文件索引
  - 专属格式，易于识别和管理
  - 可扩展到其他平台的导出工具

---

## 兼容性考虑

1. **向后兼容**: 老版本没有 .xyddjson，小遥搜索可以正常导入文件
2. **可选功能**: 用户可以删除 .xyddjson 文件，不影响原文件使用
3. **部分导出**: 只为下载的文件生成元数据

---

## 优势总结

| 优势 | 说明 |
|------|------|
| **紧密关联** | 元数据与文件同名，一目了然 |
| **易于管理** | 移动/复制文件时自然地带上元数据 |
| **简单高效** | 小遥搜索只需读取同名文件 |
| **可扩展** | 未来可支持其他平台导出 |
| **不干扰** | .xyddjson 可被配置忽略 |
