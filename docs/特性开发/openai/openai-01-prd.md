## 1. 特性概述

### 1.1 定位（一句话）

为小遥搜索添加 OpenAI 兼容的大语言模型服务支持，用户可选择使用云端大模型（如阿里云通义千问、DeepSeek、Moonshot 等）替代或补充本地 Ollama 模型。

### 1.2 背景与动机

**用户痛点**：
- 本地 Ollama 需要较好的硬件配置，对低配置用户不友好
- 本地模型能力有限，复杂任务效果不佳
- 云端大模型成本持续下降，性价比提升
- 用户希望有更多选择，根据场景灵活切换

**业务价值**：
- 降低使用门槛：无需本地 GPU 即可使用强大模型
- 提升搜索质量：云端大模型能力更强，查询增强效果更好
- 增加用户粘性：云端 API 密钥绑定增强平台依赖
- 扩展商业模式：为后续云服务订阅奠定基础

**技术必要性**：
- 现有代码已预留云端备选配置，但缺少实现
- OpenAI API 已成为事实标准，多数供应商兼容
- 统一的服务接口便于后续扩展

### 1.3 目标用户

**主要用户**：
- 硬件配置较低的个人用户
- 需要更强模型能力的知识工作者
- 希望降低运维成本的企业用户

**使用场景**：
- 场景1：个人笔记本用户，使用阿里云通义千问进行查询增强
- 场景2：企业用户，使用 DeepSeek 进行批量文档处理
- 场景3：开发者，使用 Moonshot 进行 API 测试

### 1.4 与主版本的关系

**依赖关系**：
- 依赖主 PRD 中的"大语言模型查询增强"功能
- 依赖现有的 AI 模型管理框架
- 依赖配置管理 API

**影响范围**：
- 后端：新增 OpenAI 兼容服务类，修改模型管理器
- 前端：设置页面增加类型选择和动态表单
- 数据库：无变更（复用现有 ai_models 表）

## 4. 核心流程（增量部分）

### 4.1 新增流程

```
用户配置云端大模型流程：

1. 打开设置页面 → 大语言模型标签
2. 选择模型类型：OpenAI 兼容（云端）
3. 填写配置信息：
   - API 密钥（必填）
   - 端点地址（可选，有默认值）
   - 模型名称（必填）
4. 点击测试连接验证配置
5. 点击保存设置
6. 重启应用生效
```

### 4.2 修改流程

```
原有流程：
设置 → 选择 Ollama → 配置模型名称和地址 → 保存

修改后流程：
设置 → 选择模型类型 → 根据类型显示对应表单 → 测试连接 → 保存
                     ↑
                 类型选择器（新增）
          Ollama（本地） | OpenAI 兼容（云端）
```

## 7. 技术方案

### 7.1 技术选型

| 技术/框架 | 用途 | 选择理由 | 替代方案 |
|----------|------|---------|---------|
| aiohttp | HTTP 客户端 | 异步高性能，已在项目中使用 | httpx |
| Pydantic | 配置验证 | 类型安全，自动验证 | 手动验证 |
| BaseAIModel | 服务基类 | 统一接口，易于扩展 | 独立实现 |

### 7.2 架构变更

**前端架构**：
```
Settings.vue
├── llmConfig (reactive)
│   ├── provider: 'local' | 'cloud'    // 新增
│   ├── model_name: string
│   ├── base_url: string                // local 使用
│   ├── api_key: string                 // cloud 使用
│   ├── endpoint: string                // cloud 使用
│   └── ...
└── saveLLMConfig()
    └── 根据 provider 构建不同请求体
```

**后端架构**：
```
app/services/
├── ai_model_base.py              # 基类
│   └── BaseAIModel (ABC)
├── ollama_service.py             # Ollama 本地服务
│   └── OllamaLLMService extends BaseAIModel
└── openai_llm_service.py         # OpenAI 兼容服务（新增）
    └── OpenAILLMService extends BaseAIModel

app/services/ai_model_manager.py
└── register_model()
    └── 根据 provider 创建对应服务实例
        - provider='local' → OllamaLLMService
        - provider='cloud' → OpenAILLMService
```

**数据流**：
```
前端选择类型 → 提交配置 → 后端解析 provider
                              ↓
                    创建对应服务实例
                              ↓
                    测试连接 / 加载模型
```

### 7.3 依赖变更

**后端依赖**：
```
# 无新增依赖，复用现有 aiohttp
aiohttp>=3.9.0
```

### 18.3 FAQ

**Q1: 云端模型和本地模型可以同时使用吗？**
A: 不可以。两者是互斥的，用户只能选择其中一种。

**Q2: API 密钥会被上传到小遥搜索服务器吗？**
A: 不会。API 密钥只存储在本地，加密保存，直接调用对应供应商的 API。

**Q3: 支持哪些云服务供应商？**
A: 所有兼容 OpenAI API 标准的供应商，包括 OpenAI、阿里云通义千问、DeepSeek、Moonshot 等。

**Q4: 如何切换供应商？**
A: 只需修改端点地址和模型名称即可，配置格式一致。
