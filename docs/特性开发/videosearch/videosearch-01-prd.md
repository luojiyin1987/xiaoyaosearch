## 1. 特性概述

### 1.1 定位（一句话）
为小遥搜索添加视频画面搜索能力，用户可通过图片检索视频内容，快速定位视频中的关键画面。

### 1.2 背景与动机

**用户痛点**：
- **痛点1**：用户保存了大量视频文件，但无法通过画面内容快速搜索，只能靠记忆或手动浏览
- **痛点2**：内容创作者需要从大量素材中找到特定画面，现有基于音频转录的搜索无法满足需求
- **痛点3**：知识工作者录制了大量教学/会议视频，但难以定位某个知识点的具体位置

**业务价值**：
- 扩展搜索能力覆盖视频内容，提升产品竞争力
- 弥补现有视频搜索仅基于音频转录的不足
- 为内容创作者和知识工作者提供更高效的检索工具

**技术必要性**：
- 项目已实现CN-CLIP图像搜索能力，可复用于视频关键帧搜索
- 视频文件已纳入索引范围，需补充画面维度的搜索能力
- 配置系统成熟，支持灵活的功能开关和参数配置

### 1.3 目标用户
- **主要用户**：内容创作者（视频博主、UP主）、知识工作者（教师、学生、研究人员）
- **使用场景**：
  - UP主从素材库中找到某个表情/动作的画面
  - 教师从课程视频中找到某个知识点的讲解位置
  - 用户从录制的会议视频中找到某页PPT展示的画面

### 1.4 与主版本的关系
- **依赖关系**：
  - 依赖现有的CN-CLIP图像搜索能力（[chunk_index_service.py:1682-1794](backend/app/services/chunk_index_service.py)）
  - 依赖配置管理系统（[config.py](backend/app/core/config.py)、[settings_service.py](backend/app/services/settings_service.py)）
  - 依赖前端图片输入界面（[Home.vue:364-429](frontend/src/renderer/src/views/Home.vue)）
- **影响范围**：
  - 后端：新增关键帧提取服务、扩展索引构建逻辑、新增配置项
  - 前端：搜索结果展示扩展
  - 数据库：新增视频帧元数据表

## 4. 核心流程（增量部分）

### 4.1 新增流程

**视频画面搜索流程**：
```
用户上传图片
    ↓
前端调用 /api/search/multimodal（复用现有接口）
    ↓
后端使用CN-CLIP模型提取图片特征向量
    ↓
在Faiss图像索引中搜索（包含图片+视频关键帧）
    ↓
返回匹配结果（包含图片和视频画面）
    ↓
前端展示结果，视频画面显示时间戳
```

**关键帧提取与索引流程**：
```
索引构建触发
    ↓
遍历视频文件（仅当视频画面搜索启用时）
    ↓
对每个视频提取关键帧：
  - 从0秒开始，按配置间隔提取（如每10秒）
  - 提取到配置的时长上限（如前10分钟）
  - 保存临时图片文件
    ↓
使用CN-CLIP模型提取关键帧特征向量
    ↓
将特征向量添加到Faiss图像索引
    ↓
保存元数据：video_id, timestamp, vector_id
    ↓
删除临时关键帧图片文件
```

### 4.2 修改流程

**图像索引构建流程变更**：
```
原有步骤：遍历图片文件 → 提取特征 → 构建索引
               ↓
修改为：遍历图片文件 → 提取特征 → 构建索引
       ↓
   [新增] 遍历视频文件 → 提取关键帧 → 提取特征 → 添加到索引
                       ↑
                   变更点说明：
                   - 关键帧临时提取，用完即删
                   - 视频帧元数据单独存储
                   - 支持配置开关控制是否执行
```

### 5.2 后端设计

#### 配置文件变更
**新增配置类**：`VideoFrameConfig`（在 `backend/app/core/config.py` 中）
```python
class VideoFrameConfig(BaseSettings):
    """视频画面搜索配置"""
    # 功能开关
    video_frame_search_enabled: bool = Field(
        default=False,
        description="是否启用视频画面搜索"

    # 关键帧提取配置
    video_frame_interval: int = Field(
        default=10,
        ge=5,

        default=10,
        ge=1,
        le=60,
        description="视频提取时长上限（分钟）"
    )
```

**配置方式**：
- 通过 `.env` 文件或环境变量配置

  - `VIDEO_FRAME_INTERVAL=10`

#### API变更
| 接口 | 变更类型 | 变更内容 | 向后兼容 |
|-----|---------|---------|---------|
| /api/search/multimodal | 修改 | 响应中增加视频画面匹配结果 | 是 |

**修改接口详情**：
```
原接口：POST /api/search/multimodal
变更点：
- 响应字段新增：
  - video_results: VideoFrameResult[] - 视频画面匹配结果
  - VideoFrameResult结构：
    - file_id: int - 视频文件ID
    - file_name: string - 视频文件名
    - file_path: string - 视频文件路径
    - timestamp: int - 匹配时间戳（秒）
    - similarity: float - 相似度分数
    - frame_preview: string - 关键帧预览图（可选）
```

#### 数据库变更

**新增表**：`video_frames`
```sql
CREATE TABLE video_frames (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    video_id INTEGER NOT NULL,           -- 关联的视频文件ID
    timestamp INTEGER NOT NULL,           -- 时间戳（秒）
    vector_id INTEGER NOT NULL,           -- Faiss索引中的向量ID
    file_id INTEGER NOT NULL,             -- 关联files表ID
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(video_id, timestamp)
);

CREATE INDEX idx_video_frames_video_id ON video_frames(video_id);
CREATE INDEX idx_video_frames_vector_id ON video_frames(vector_id);
```

#### 业务逻辑变更

**新增逻辑**：
1. **关键帧提取服务**（`backend/app/services/video_frame_extractor.py`）：
   - 使用OpenCV或FFmpeg提取视频关键帧
   - 支持可配置的提取间隔和时长上限
   - 返回临时图片文件路径和时间戳列表

2. **视频帧元数据管理**（`backend/app/models/video_frame.py`）：
   - 定义视频帧数据模型
   - 提供CRUD操作

**修改逻辑**：
1. **索引构建服务**（`backend/app/services/chunk_index_service.py`）：
   - `_build_clip_image_index()` 方法扩展：
     - 检查 `video_frame_search_enabled` 配置
     - 如果启用，遍历视频文件并提取关键帧
     - 将关键帧特征向量添加到CLIP索引
     - 保存视频帧元数据到数据库
   - 元数据文件扩展：包含视频帧信息

2. **多模态搜索服务**（`backend/app/api/search.py`）：
   - `/api/search/multimodal` 接口响应扩展：
     - 根据向量ID查询是否为视频帧
     - 如果是视频帧，查询 `video_frames` 表获取详细信息
     - 返回视频画面匹配结果

**删除逻辑**：无

## 7. 技术方案

### 7.1 技术选型
| 技术/框架 | 用途 | 选择理由 | 替代方案 |
|----------|------|---------|---------|
| OpenCV (cv2) | 视频关键帧提取 | 成熟稳定、Python生态好、性能优秀 | FFmpeg-python、MoviePy |
| CN-CLIP (现有) | 图像特征编码 | 已集成、中文优化、与图搜图一致 | OpenAI CLIP |
| Faiss (现有) | 向量相似度搜索 | 已集成、性能优异、与图搜图一致 | Chroma、Weaviate |
| SQLite (现有) | 元数据存储 | 已集成、轻量、适合本地应用 | PostgreSQL、MySQL |

### 7.2 架构变更
- **前端架构**：无重大变更，复用现有搜索流程，仅扩展搜索结果展示
- **后端架构**：
  - 新增 `VideoFrameConfig` 配置类（config.py）
  - 新增 `VideoFrameExtractor` 服务
  - 扩展 `ChunkIndexService` 索引构建流程
  - 扩展 `MultimodalSearchAPI` 响应结构
- **数据流**：
  - 索引构建：视频文件 → 关键帧提取 → CN-CLIP编码 → Faiss索引 + SQLite元数据
  - 搜索流程：图片查询 → CN-CLIP编码 → Faiss搜索 → 合并图片+视频帧结果
- **配置管理**：
  - 通过 .env 文件或环境变量配置
  - 配置项：VIDEO_FRAME_SEARCH_ENABLED、VIDEO_FRAME_INTERVAL、VIDEO_FRAME_MAX_DURATION

### 18.3 FAQ

**Q1: 为什么关键帧是临时提取的？**
A: 关键帧仅用于提取特征向量，向量存储在Faiss索引中。临时提取可以节省磁盘空间，避免存储大量图片文件。

**Q2: 能否搜索视频的任意位置？**
A: 当前版本仅支持搜索前N分钟（可配置）的关键帧。完整视频搜索需要提取更多帧，会显著增加索引构建时间和索引大小，计划在后续版本中优化。

**Q3: 支持哪些视频格式？**
A: 支持现有索引系统中的所有视频格式：mp4、avi、mkv、mov、wmv、flv、webm。

**Q4: 视频画面搜索会影响图片搜索的性能吗？**
A: 影响很小。视频关键帧与图片使用同一个Faiss索引，搜索时间主要取决于索引大小。由于只提取前N分钟的关键帧，索引增长可控。
