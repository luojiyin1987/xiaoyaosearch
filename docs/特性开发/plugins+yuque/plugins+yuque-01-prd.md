## 1. 特性概述

### 1.1 定位（一句话）
为小遥搜索建立插件化架构，支持多数据源扩展，优先实现语雀知识库数据源，让用户能够搜索本地文件和云端知识库。

### 1.2 背景与动机

- **用户痛点**：
  - 痛点1：知识工作者同时使用本地文件和语雀等云端知识库，需要在多个平台间切换查找内容
  - 痛点2：团队文档分散在语雀中，无法与本地文件一起进行全局语义搜索
  - 痛点3：第三方开发者无法独立开发和贡献新的数据源插件

- **业务价值**：
  - 扩展产品边界，从本地搜索升级为全场景知识搜索
  - 提升用户粘性，云端数据源绑定增强平台依赖
  - 建立插件生态，降低后续数据源接入成本

- **技术必要性**：
  - 当前数据源耦合在核心代码中，扩展需要修改核心逻辑
  - 缺少统一的数据源抽象层，重复开发成本高
  - 无热插拔能力，新增数据源需要重启服务

### 1.3 目标用户

- **主要用户**：
  - 使用语雀进行团队协作的知识工作者
  - 技术开发者（希望贡献数据源插件）
  - 运维人员（通过API配置和管理数据源）

- **使用场景**：
  - 产品经理搜索本地产品文档 + 语雀需求文档
  - 开发者搜索本地代码 + 语雀技术文档
  - 运维人员通过API配置语雀数据源并同步文档

### 1.4 与主版本的关系

- **依赖关系**：
  - 依赖主PRD中的多模态搜索功能（文本、语音、图像输入）
  - 依赖主PRD中的索引构建服务（Faiss + Whoosh）
  - 依赖主PRD中的AI模型集成（BGE-M3、CN-CLIP）

- **影响范围**：
  - 后端新增插件框架模块
  - API层新增插件管理接口
  - 前端新增数据源配置界面
  - 搜索结果支持多数据源标识

## 4. 核心流程（增量部分）

### 4.1 新增流程

**流程1：插件安装与配置**
```
用户将插件文件放到插件目录 → 系统启动时自动发现和加载插件
                                              ↓
                                    读取插件配置文件(config.yaml)
                                              ↓
                                    初始化插件连接 → 自动开始数据同步
                                              ↓
                                    构建索引 → 搜索结果包含该数据源
```

**流程2：语雀文档同步**
```
触发同步任务 → 调用语雀API → 获取文档列表
                                  ↓
                            遍历文档内容 → 标准化为DataSourceItem
                                  ↓
                            提交索引服务 → Faiss+Whoosh构建索引
                                  ↓
                            更新同步状态 → 完成同步
```

**流程3：多数据源搜索**
```
用户发起搜索 → 搜索服务接收 → 查询所有启用的数据源
                                      ↓
                              Faiss向量检索 → Whoosh全文检索
                                      ↓
                              聚合搜索结果 → 按数据源标识分类
                                      ↓
                              返回前端展示 → 用户点击跳转原文
```

### 4.2 修改流程

**修改：索引构建流程**
```
原流程：扫描本地文件 → 解析内容 → 构建索引
新流程：数据源管理器 → 扫描各数据源 → 标准化数据项 → 构建索引
                              ↑
                         语雀插件/本地文件/未来插件
```

## 5. 详细设计

### 5.1 后端设计

#### API变更

| 接口 | 变更类型 | 变更内容 | 向后兼容 |
|-----|---------|---------|---------|
| /api/search | 修改 | 响应增加source_type、source_url、source_name字段 | 是 |

**修改接口详情**：

```
原接口：POST /api/search
变更点：
- 新增字段：source_type - 数据源类型（filesystem/yuque/feishu等）
- 新增字段：source_url - 原文访问链接
- 新增字段：source_name - 数据源显示名称
```

**插件配置方式说明**：
插件通过放置到指定目录并编辑配置文件来管理，无需API接口。
- 插件目录：`data/plugins/{plugin_id}/`
- 配置文件：`data/plugins/{plugin_id}/config.yaml`
- 插件文件：`data/plugins/{plugin_id}/plugin.py`

**修改接口详情**：

```
原接口：POST /api/search
变更点：
- 新增字段：source_type - 数据源类型（filesystem/yuque/feishu等）
- 新增字段：source_url - 原文访问链接
- 新增字段：source_name - 数据源显示名称
```

#### 数据库变更

| 表名 | 变更类型 | 变更内容 | SQL示例 |
|-----|---------|---------|---------|
| files | 修改表 | 新增source_type、source_url字段 | ALTER TABLE files ADD COLUMN... |

```sql
-- 为files表新增数据源字段
ALTER TABLE files ADD COLUMN source_type TEXT DEFAULT 'filesystem';
ALTER TABLE files ADD COLUMN source_url TEXT;

-- 创建索引
CREATE INDEX idx_files_source_type ON files(source_type);
```

**数据库设计说明**：
- 不需要独立的插件元数据表，插件通过文件系统管理
- 不需要插件配置表，配置通过YAML文件管理
- 复用现有files表存储索引数据，通过source_type区分数据源

#### 业务逻辑变更

- **新增逻辑**：
  - 插件自动发现和加载机制（启动时扫描插件目录）
  - 插件配置文件解析（YAML格式）
  - 多数据源扫描和标准化
  - 多数据源索引聚合

- **修改逻辑**：
  - FileScanner作为默认数据源内置
  - 搜索服务增加数据源过滤和标识
  - 索引服务支持多数据源混合索引

- **删除逻辑**：
  - 移除硬编码的数据源扫描逻辑
  - 无需插件管理API相关逻辑

## 7. 技术方案

### 7.1 技术选型

| 技术/框架 | 用途 | 选择理由 | 替代方案 |
|----------|------|---------|---------|
| importlib | 插件动态加载 | Python标准库，无额外依赖 | stevedore |
| ABC | 插件接口定义 | 标准抽象基类，类型安全 | Protocol |
| Pydantic | 配置验证 | 已有依赖，schema验证 | JSON Schema |
| httpx | 语雀API调用 | 异步HTTP客户端，已集成 | aiohttp |
| YAML | 插件配置 | 人类可读，易于编辑 | JSON/TOML |

### 7.2 架构变更

- **前端架构**：
  - 新增 `datasource/` 模块，包含数据源列表、配置、状态组件
  - 使用 Pinia 管理数据源状态

- **后端架构**：
  - 新增 `plugins/` 模块，包含接口定义和管理器
  - 新增 `services/datasource/` 模块，包含数据源抽象
  - 修改 `services/file_index_service.py` 支持多数据源

- **数据流**：
  ```
  语雀API → YuqueDataSource.scan() → DataSourceItem
                                          ↓
                            UnifiedIndexService
                                          ↓
                            Faiss + Whoosh
  ```

### 7.3 依赖变更

**后端依赖**：
```python
# 无新增依赖，使用现有依赖
# pydantic>=2.0 - 已有
# httpx>=0.24 - 已有
# pyyaml>=6.0 - 新增，用于插件配置
```

### 18.3 FAQ

**Q1: 如何配置语雀数据源？**
A: 将语雀插件放到data/plugins/yuque/目录，编辑config.yaml文件填写API Token和知识库ID

**Q2: 语雀API Token在哪里获取？**
A: 登录语雀后，访问 账户设置 -> 个人访问令牌 -> 新建令牌

**Q3: 插件会访问我的语雀隐私数据吗？**
A: 插件只会读取您配置的知识库文档内容，不会访问其他数据，且所有数据仅存储在本地

**Q4: 可以同时使用多个数据源吗？**
A: 可以，将多个插件放到插件目录即可，系统会自动加载所有插件，搜索时会聚合所有数据源的结果

**Q5: 插件同步会影响系统性能吗？**
A: 同步任务在后台异步执行，不会阻塞搜索功能

**Q6: 如何开发自定义数据源插件？**
A: 参考插件开发规范，实现DataSourcePlugin接口，将插件文件放到data/plugins/目录即可
