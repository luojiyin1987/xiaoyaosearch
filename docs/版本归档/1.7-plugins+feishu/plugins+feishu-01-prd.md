## 1. 特性概述

### 1.1 定位（一句话）
为小遥搜索添加飞书文档数据源识别能力，让用户能够搜索从飞书导出的本地 Markdown 文档。

### 1.2 背景与动机

- **用户痛点**：
  - 痛点1：团队文档存储在飞书中，用户需要定期导出到本地进行搜索
  - 痛点2：导出的飞书文档与本地文件混合存储，无法区分数据来源
  - 痛点3：搜索到飞书文档后，无法快速跳转到飞书原文链接

- **业务价值**：
  - 降低使用门槛：用户无需额外配置，导出文件即可被识别
  - 提升搜索体验：在搜索结果中清晰标识飞书文档来源
  - 快速定位原文：支持从搜索结果直接跳转到飞书原文

- **技术必要性**：
  - 基于已有插件化架构，复用数据源识别机制
  - 飞书导出格式标准化，可通过元数据解析实现识别

### 1.3 目标用户

- **主要用户**：
  - 使用飞书进行团队协作的知识工作者
  - 需要将飞书文档与本地文件一起搜索的用户

- **使用场景**：
  - 产品经理搜索本地产品文档 + 飞书需求文档
  - 开发者搜索本地代码 + 飞书技术文档
  - 用户搜索后通过原文链接快速定位飞书文档

### 1.4 与主版本的关系

- **依赖关系**：
  - 依赖主PRD中的多模态搜索功能
  - 依赖主PRD中的索引构建服务
  - 依赖插件化架构（plugins+yuque特性）

- **影响范围**：
  - 后端新增飞书数据源识别插件
  - 文件扫描服务支持飞书元数据解析
  - 搜索结果支持飞书数据源标识（前端已支持）

## 5. 详细设计

### 5.1 飞书导出格式规范

飞书导出的 Markdown 文档在文件末尾包含以下元数据格式：

```markdown
---
> 更新: 2026-03-30 02:52:46
> 来源类型: feishu
> 原文: <https://feishu.cn/wiki/MZKMwqpljiod1ak38Cscnr8hnkh>
---
```

**元数据字段说明**：
- `更新`：文档最后更新时间
- `来源类型`：固定为 `feishu`
- `原文`：飞书文档的完整 URL

#### 元数据解析逻辑

**正则表达式模式**：

```python
# 飞书元数据检测模式
FEISHU_METADATA_PATTERNS = [
    # 标准格式
    r'>\s*来源类型:\s*feishu.*?>\s*原文:\s*<(https://[^\s>]+)>',
    # 兼容格式
    r'>\s*Source:\s*feishu.*?>\s*URL:\s*<(https://[^\s>]+)>',
]
```

**解析流程**：
1. 读取文件内容最后 500 字符
2. 使用正则表达式匹配飞书元数据
3. 提取原文链接
4. 返回数据源信息

#### 业务逻辑变更

**文件扫描服务修改**：

在 `FileScanner` 中集成飞书元数据检测：

```python
async def scan_file(self, file_path: str) -> FileInfo:
    """扫描单个文件"""
    # 读取文件内容
    content = await self._read_file_content(file_path)

    # 检测飞书元数据
    feishu_plugin = self._plugin_manager.get_plugin("feishu")
    source_info = feishu_plugin.get_file_source_info(file_path, content)

    # 设置数据源信息
    file_info.source_type = source_info.get("source_type") or "filesystem"
    file_info.source_url = source_info.get("source_url")

    return file_info
```

### 18.3 FAQ

**Q1: 如何使用飞书文档搜索？**
A: 将飞书文档导出为 Markdown 格式，放到小遥搜索的扫描目录即可，系统会自动识别。

**Q2: 飞书导出格式是什么？**
A: 飞书导出的 Markdown 文件末尾包含元数据块，标明来源类型和原文链接。

**Q3: 可以搜索飞书文档中的图片吗？**
A: 目前支持文本搜索，图片搜索需要飞书文档导出时包含图片。

**Q4: 飞书原文链接会过期吗？**
A: 如果您有飞书文档的访问权限，原文链接可以正常打开。
