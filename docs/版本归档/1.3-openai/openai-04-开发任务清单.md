## 设计原则：类型互斥 + 统一接口 + 向后兼容

本特性采用**类型互斥**设计，用户只能选择 Ollama（本地）或 OpenAI 兼容（云端）其中一种：
- **类型互斥**：前端配置表单互斥显示，后端根据 provider 创建对应服务
- **统一接口**：两种服务继承同一基类 `BaseAIModel`
- **向后兼容**：精简 Ollama 服务，移除云端备选逻辑
- **安全优先**：API 密钥加密存储，日志脱敏处理
